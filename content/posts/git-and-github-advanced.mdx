## ভূমিকা

প্রোগ্রামিং বা সফটওয়্যার ইঞ্জিনিয়ারিংয়ের জগতে **Git** এবং **GitHub** হলো অক্সিজেন। এটি ছাড়া টিম ওয়ার্ক এবং কোড ম্যানেজমেন্ট প্রায় অসম্ভব। চলুন খুব সহজ ভাষায় এবং ভিজ্যুয়ালের সাহায্যে পুরো ব্যাপারটা বুঝে নিই।

---

## ১. Git কী?

Git হলো একটি **Version Control System (VCS)**। সহজ ভাষায়, এটি আপনার কোডের একটি টাইমমেশিন। আপনি কখন, কোথায়, কী কোড লিখেছেন, তার পুরো হিস্ট্রি গিট মনে রাখে। যদি কোনো কারণে কোড নষ্ট হয়ে যায়, আপনি চাইলেই আগের ভার্সনে ফিরে যেতে পারবেন।

## ২. GitHub কী?

Git হলো আপনার কম্পিউটারের লোকাল সফটওয়্যার, আর GitHub হলো ক্লাউড স্টোরেজ (যেমন Google Drive) যেখানে আপনি আপনার গিট রিপোজিটরি (প্রজেক্ট) অনলাইনে হোস্ট করে রাখতে পারেন।

---

## ৩. Git এর প্রধান ৪টি স্টেজ (Visual Workflow)

আপনার কোড গিটহাবে পৌঁছানোর আগে সাধারণত ৪টি স্টেপ পার করে:

```text
 💻 Working Directory   👉   📦 Staging Area   👉   🏠 Local Repository   👉   ☁️ Remote (GitHub)
     (আপনার পিসি)              (প্রস্তুতি)                 (কমিট)                    (পুশ)
         │                         │                          │                         │
         ├──── git add ───────────>│                          │                         │
         │                         ├───── git commit ────────>│                         │
         │                         │                          ├───── git push ─────────>│
```

### স্টেপগুলো বিস্তারিত:
1. **Working Directory:** আপনি ভিএস কোডে (VS Code) বা এডিটরে যে ফাইলগুলো নিয়ে কাজ করছেন।
2. **Staging Area (`git add`):** আপনার কোড কমিট করার আগে একটি নির্দিষ্ট জায়গায় জমা হয়। এটি একটি চেকপয়েন্ট।
3. **Local Repository (`git commit`):** কোডের একটি পার্মানেন্ট স্ন্যাপশট আপনার পিসিতে সেভ হলো।
4. **Remote Repository (`git push`):** পিসির কোডটি ইন্টারনেটে (GitHub) আপলোড করা হলো।

## ৪. দরকারি কমান্ডসমূহ

### 🔹 প্রজেক্ট শুরু করা (Initialize)
নতুন প্রজেক্টে গিট চালু করতে:
```bash
git init
```

### 🔹 কোড স্টেজিং এরিয়ায় নেওয়া
```bash
git add index.html  # একটি ফাইল অ্যাড করতে
git add .           # সব ফাইল অ্যাড করতে
```

### 🔹 কোড সেভ করা (Commit)
```bash
git commit -m "Added index.html and style.css"
```

### 🔹 বর্তমান অবস্থা চেক করা
```bash
git status
```

### 🔹 হিস্ট্রি দেখা
```bash
git log
```

## ৫. ব্রাঞ্চিং (Branching)

ব্রাঞ্চিং হলো গিটের সবচেয়ে পাওয়ারফুল ফিচার। ধরুন, আপনার একটি মেইন ওয়েবসাইট আছে। আপনি নতুন একটি ফিচার বানাতে চান কিন্তু মেইন কোড নষ্ট করতে চান না। তখন আপনি একটি নতুন 'Branch' তৈরি করে কাজ করবেন।

```text
(main branch)    A ─── B ─── C ─── F (merge)
                              \   /
(new-feature)                  D ─ E
```

### ব্রাঞ্চ কমান্ডসমূহ:
```bash
git branch              # সব ব্রাঞ্চ দেখতে
git branch new-feature  # নতুন ব্রাঞ্চ খুলতে
git checkout new-feature # নতুন ব্রাঞ্চে যেতে
# অথবা শর্টকাট:
git checkout -b new-feature
```

## ৬. GitHub এর সাথে যুক্ত করা

আপনার লোকাল কোড গিটহাবে পাঠাতে:

```bash
# গিটহাবের রিমোট লিংক অ্যাড করা
git remote add origin https://github.com/your-username/repo-name.git

# কোড পুশ করা
git push -u origin main
```

---

## ৭. Pull এবং Clone

অন্যের কোড বা আপনার নিজের গিটহাবের কোড পিসিতে নামাতে:

```bash
# পুরো প্রজেক্ট পিসিতে ডাউনলোড করতে
git clone https://github.com/user/repo.git

# গিটহাব থেকে নতুন আপডেট পিসিতে আনতে
git pull origin main
```
