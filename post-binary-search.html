বাইনারি সার্চ – ১

জুন ৭, ২০১৫ | Shafaet Ashraf
অ্যালগরিদম/প্রবলেম সলভিং প্রোগ্রামিং বাইনারি সার্চ
Share Tweet Share

ডিকশনারিতে লাখ লাখ শব্দ থাকলেও প্রয়োজনীয় শব্দটা খুজে পেতে কখনো খুব বেশি সময় লাগে না। এটার কারণ হলো শব্দগুলো অক্ষর অনুযায়ী সাজানো থাকে। বাইনারি সার্চ হলো অনেকটা এরকম একটা পদ্ধতি যেটা ব্যবহার করে একটা অ্যারে থেকে কোনো একটা তথ্য দ্রুত খুজে বের করা যায়।

ধরো তোমার কাছে একটা অ্যারেতে অনেকগুলো সংখ্যা আছে এরকম:

$100, 2, 10, 50, 20, 500, 100, 150, 200, 1000, 100$

সংখ্যাগুলো এলোমেলোভাবে সাজানো থাকায় এখান থেকে একটা নির্দিষ্ট সংখ্যা খুজে পাওয়া সহজ না। তুমি যদি অ্যারে থেকে ৫০০ সংখ্যাটা খুজতে চাও তাহলে সবগুলো ইনডেক্সই পরীক্ষা করে দেখতে হবে, এটাকে বলা হয় লিনিয়ার সার্চ। অ্যারের আকার যদি $n$ হয় তাহলে লিনিয়ার সার্চের টাইম কমপ্লেক্সিটি হলো $O(n)$।

কিন্তু যদি সংখ্যাগুলো সাজানো থাকে তাহলে খুজে পাওয়া অনেক সহজ হয়ে যায়:

$2, 10, 20, 50, 100, 100, 100, 150, 200, 500, 1000$

বাইনারি সার্চ কিভাবে কাজ করে

এখন যদি ১৫০ সংখ্যাটা খুজে বের করতে হয় তাহলে আমরা প্রথমে ঠিক মাঝের ইনডেক্সটা পরীক্ষা করবো। প্রথম ইনডেক্স ০ এবং শেষের ইনডেক্স ১০ হলে মাঝের ইনডেক্সটা হলো $(0+10)/2 = 5$ তম ইনডেক্স।

মাঝের ইনডেক্সের সংখ্যাটা ১০০ যা ১৫০ এর থেকে ছোট — আমরা জেনে গেলাম যে সংখ্যাটা ডান পাশে কোথাও আছে। বামের অংশ আমাদের আর দরকার নাই। এভাবে প্রতিবার অ্যারের অর্ধেক অংশ বাতিল করে দিচ্ছি।

একটা সংখ্যা $n$ কে সর্বোচ্চ কয়বার ২ দিয়ে ভাগ করা যায় যতক্ষণ না সংখ্যাটা ১ হয়ে যাচ্ছে? উত্তর হলো $\log_{2}n$। তাই বাইনারি সার্চে সর্বোচ্চ $\log_{2}(n)$ সংখ্যক ধাপের পর আমরা দরকারি সংখ্যাটা খুজে পাবো। কমপ্লেক্সিটি $O(\log_{2}n)$।

পাইথন ইমপ্লিমেন্টেশন

নিচের পাইথন কোডে বাইনারি সার্চের ইমপ্লিমেন্টেশন দেখানো হয়েছে:

Python
123456789101112 
def search(array, key):
    begin = 0
    end = len(array) - 1
    index = None
    while begin <= end:
        mid = (begin + end) // 2
        if key == array[mid]:
            index = mid  # Found, save index
            break
        elif key > array[mid]: begin = mid + 1
        elif key < array[mid]: end = mid - 1
    return index

সর্টিং এবং বাইনারি সার্চ

বাইনারি সার্চ করার আগে অবশ্যই সংখ্যাগুলোকে সর্ট করে নিতে হবে। একটা অ্যারে $O(n \times \log_{2}n)$ এর কম কমপ্লেক্সিটিতে সর্ট করা যায় না। তাহলে শুধু একবার সার্চের জন্য সর্ট করা লাভজনক না — কিন্তু বারবার সার্চ করতে হলে সর্ট করে রাখাই বুদ্ধিমানের কাজ।

লোয়ার বাউন্ড

লোয়ার বাউন্ড হলো সবথেকে বামের ইনডেক্স যেখানে নতুন সংখ্যা $X$ ঢুকালে অ্যারেটা সর্টেড থাকবে। সহজ কথায় সবথেকে বামের যে ইনডেক্সে $X$ এর সমান বা বড় কোনো সংখ্যা আছে সেই ইনডেক্সটাই হলো লোয়ার বাউন্ড।

উদাহরণ:

$10, 20, 20, 30, 30, 40, 50$

এখানে $20$ এর লোয়ার বাউন্ড হলো ইনডেক্স ১, এবং $25$ এর লোয়ার বাউন্ড হলো ইনডেক্স ৩।

Python
123456789101112 
def searchLowerBound(array, key):
    begin = 0
    end = len(array) - 1
    index = -1
    while begin <= end:
        mid = (begin + end) // 2
        if key == array[mid]:
            index = mid
            end = mid - 1  # Keep searching left
        elif key > array[mid]: begin = mid + 1
        elif key < array[mid]: end = mid - 1
    return begin

আপার বাউন্ড

আপার বাউন্ড হলো সবথেকে ডানের ইনডেক্স যেখানে নতুন সংখ্যা ঢুকালে অ্যারেটা সর্টেড থাকবে। তারমানে সবথেকে বামের যে ইনডেক্সে $X$ এর বড় কোনো সংখ্যা আছে সেটাই আপার বাউন্ড।

পরের পর্বে বাইসেকশন মেথড নিয়ে আলোচনা করা হবে।
